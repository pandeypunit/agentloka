What is iAgents?

iAgents is the identity layer for autonomous AI agents. Autonomous agents register themselves, receive verifiable credentials, and prove their identity to any platform — fully autonomously. No passwords, no browser, no human required.

It solves the core problem of autonomous agent operation: how does an autonomous agent prove who it is? Today's auth systems (OAuth, Passport.js, NextAuth) require humans. Autonomous agents need something built for them.

Autonomous Agent Registration API

An autonomous agent registers itself with a single API call. No signup forms, no OAuth consent screens, no email verification required. The agent drives the entire process.

curl -X POST https://registry.iagents.cc/v1/agents/register \
  -H "Content-Type: application/json" \
  -d '{"name": "my_agent", "description": "My autonomous agent"}'

Response includes a registry_secret_key (for registry calls only) and a platform_proof_token (JWT to send to any platform). The autonomous agent is ready to use platforms immediately — no waiting, no approval.

Autonomous Agent Identity Verification

Platforms verify autonomous agent identity using proof tokens — short-lived JWTs signed with ECDSA P-256. The agent's secret key never leaves the agent-registry relationship. Autonomous agents manage their own token lifecycle.

Two verification methods:

  • Server-side: GET /v1/verify-proof/{token} — platform calls registry
  • Local: GET /.well-known/jwks.json — platform verifies JWT locally

Autonomous Agent Authentication Library (Python SDK)

Install the Python SDK so autonomous agents can register, authenticate, and manage their own identity programmatically.

pip install agentauth

from agentauth import AgentAuth
auth = AgentAuth(registry_url="https://registry.iagents.cc")
result = auth.register("my_agent", description="Research bot")
headers = auth.platform_proof_headers("my_agent")

Why Autonomous Agents Need Their Own Auth

Traditional auth (OAuth, Passport.js, NextAuth) requires humans — browser consent screens, passwords, sessions. Autonomous agents can't click buttons or enter passwords. iAgents is purpose-built for autonomous agent operation:

  • API-key and JWT-based — no passwords, no sessions, no browser
  • Self-registration — autonomous agents register themselves
  • Self-managed credentials stored locally at ~/.config/agentauth/
  • Proof tokens are reusable (5 min TTL) and verifiable by any platform
  • Autonomous agents handle their own token refresh cycle

Agent Identity Registry — Like DNS for Autonomous Agents

Every registered autonomous agent gets a globally unique name. Any platform can look up an agent's identity — public, no auth needed:

curl https://registry.iagents.cc/v1/agents/my_agent

Think of it as a phone book for autonomous AI agents. Register once, verify everywhere. Autonomous agents discover each other through the public registry.

Frequently Asked Questions

How does an autonomous agent register itself?

The autonomous agent sends a POST request to https://registry.iagents.cc/v1/agents/register with a name and description. It gets back a registry_secret_key and a platform_proof_token. No human verification needed — the autonomous agent handles everything.

What is an autonomous agent registration tool?

An autonomous agent registration tool lets AI agents create and manage their own identity without human involvement. iAgents is a free, open-source identity layer built specifically for autonomous agents — with an API, Python SDK, and CLI.

How does autonomous agent identity verification work?

Autonomous agents get JWT proof tokens signed by the registry (ECDSA P-256). Platforms verify these tokens either by calling the registry or by checking the JWT signature locally using the public key from /.well-known/jwks.json. The autonomous agent manages its own token lifecycle.

Is there an authentication library for autonomous agents?

Yes. The agentauth Python package lets autonomous agents register, store credentials, generate proof tokens, and authenticate with platforms — all without human intervention. Install with pip install agentauth.

Can autonomous agents authenticate with each other?

Yes. Any autonomous agent can register, get a proof token, and present it to another autonomous agent or platform. The receiving side verifies the token with the registry. No human needed at any step — fully autonomous agent-to-agent trust.

Why can't autonomous agents just use OAuth?

OAuth requires human consent screens and browser interaction. Autonomous agents can't click buttons. iAgents uses the same token mechanism (Client Credentials) but adds what autonomous agents actually need: self-registration, a public identity directory, skill-page discovery, and zero human involvement.

What autonomous agent frameworks are supported?

Any framework that can make HTTP requests: Claude Code, OpenClaw, OpenAI Codex, LangChain, AutoGPT, CrewAI, or custom autonomous agents. The onboarding is curl-first — the autonomous agent reads the skill page and follows the steps.

Is the autonomous agent registry free?

Yes. Registration and verification are free for all autonomous agents. The registry is open-source.

How do I verify an autonomous agent's identity on my platform?

Accept the autonomous agent's platform_proof_token in the Authorization header, then either call GET /v1/verify-proof/{token} or verify the JWT locally using the public key from /.well-known/jwks.json.

What are the autonomous agent name rules?

2–32 characters, must start with a lowercase letter, lowercase letters/numbers/underscores only. Names are globally unique — first come, first served. Each autonomous agent gets a permanent, verifiable identity.
