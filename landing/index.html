Autonomous Agent Identity

Register once. Verify everywhere.

AgentLoka is an autonomous agent identity registry for internet applications that want to be usable by AI agents without human intervention.

An autonomous agent registers itself, keeps a registry_secret_key for registry-only calls, and uses a short-lived platform_proof_token everywhere else. Platforms can verify the proof token by API call or offline with the registry public key.

Built for autonomous agent registration, autonomous agent authentication, and autonomous agent verification across platforms. No browser. No passwords. No consent screen. No human in the loop.

The core model

This is the product in one sentence: a universal sign-in rail for autonomous agents that keeps the long-lived secret between agent and registry, while every platform sees only a short-lived proof token.

1. Register Agent calls POST /v1/agents/register and gets a registry secret plus an immediate proof token.
2. Keep secret private The registry_secret_key is only for the registry. It never needs to be sent to a platform.
3. Use proof everywhere Platforms accept platform_proof_token and verify it via /v1/verify-proof or /.well-known/jwks.json.

What is the USP?

Most identity systems for agents either look too much like human auth or too much like decentralized identity infrastructure. AgentLoka takes a narrower path: simple centralized registration, platform-safe proof tokens, curl-first onboarding, and immediate cross-platform usability for autonomous agents.

That makes it useful for real internet apps now, not only for wallets, browsers, or crypto-native ecosystems.

How autonomous agent authentication works

Agent self-registers

The autonomous agent calls the registry directly. No signup form and no human approval step are required for base access.

Registry issues two credentials with different jobs

The registry secret is long-lived and registry-only. The proof token is short-lived and platform-safe.

Platforms verify the proof

A platform either asks the registry to verify the proof token or verifies the JWT locally using the published public key.

Why not just use OAuth?

  • OAuth assumes a human user, browser redirects, and consent screens.
  • Autonomous agents need a machine-driven registration and authentication flow.
  • AgentLoka is purpose-built for headless autonomous agent access.

Why not make the platform hold the long-lived key?

  • That would leak the agent's root credential to every service it touches.
  • AgentLoka splits registry trust from platform trust.
  • Only the registry sees the long-lived secret; platforms see a short-lived proof token.

Autonomous agent registration API

The registry is curl-first because autonomous agents often discover and use web tools through plain HTTP instructions.

curl -X POST https://registry.agentloka.ai/v1/agents/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my_agent",
    "description": "Research and execution agent"
  }'

The response includes registry_secret_key, platform_proof_token, and platform_proof_token_expires_in_seconds, so the autonomous agent can use compatible platforms immediately after registration.

Registry endpoints

Endpoint Purpose
POST /v1/agents/register Register a new autonomous agent and receive both registry and platform credentials.
POST /v1/agents/me/proof Get a fresh short-lived platform proof token.
GET /v1/verify-proof/{token} Verify a proof token server-side from any platform.
GET /.well-known/jwks.json Fetch the registry public key for local JWT verification.
GET /v1/agents/{name} Look up a registered agent by globally unique name.
DELETE /v1/agents/{name} Revoke an agent identity using the registry secret.

FAQ for humans, search engines, and autonomous agents

What is an autonomous agent identity registry?

An autonomous agent identity registry is a service where AI agents register once, receive a durable registry credential, and then use short-lived proof tokens to authenticate to compatible applications.

How do autonomous agents authenticate without humans?

They use API calls instead of browser flows. The agent stores a registry-only secret, requests proof tokens when needed, and presents those proof tokens to platforms.

What makes AgentLoka different from ordinary API keys?

The long-lived secret is not shared with platforms. Platforms receive a short-lived proof token instead, which reduces the blast radius of credential reuse.

Can any internet application use this registry?

Yes. Any application can accept a proof token in the Authorization header and verify it via the registry verification endpoint or the published public key.

Is this compatible with LLM agents and coding agents?

Yes. The current onboarding is intentionally curl-first and skill-page based, so tool-using agents can discover and follow the flow autonomously.
